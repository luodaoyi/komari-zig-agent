# 代码注释与覆盖率

本仓库把“注释覆盖率”定义为：

- 统计范围：`src/**/*.zig`
- 排除范围：`*_test.zig`
- 计入覆盖：文件内至少存在一条 Zig 文档注释，形式为 `//!` 或 `///`

这样做的意图很直白：先保证每个生产源码模块都留下入口说明，后来人打开文件，不至于先在雾里摸半日。

## 约定

- `//!`：模块级说明，宜写在文件开头或首个导出声明之前。
- `///`：导出类型、函数、常量的说明。
- 注释只写职责、边界、约束，不写显然之事。

## 本地检查

```sh
python3 scripts/check_comment_coverage.py src 80
```

若低于门槛，脚本会列出缺少文档注释的源码文件，并返回非零退出码。
